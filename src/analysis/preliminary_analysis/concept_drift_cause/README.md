# concept_drift_cause (事前分析② SHAP 比較)

事前分析①「変化区間の存在確認」で変化が確認できた後に着手する**追加実験**用のディレクトリ。
基準変化の*原因*を、古い PR の SHAP 比較で掘り下げる（層4/5/6・OOD 除外・符号反転で読む）。

現時点では**予約のみ**。設計（design.md）と各モジュールは②着手時に作成する。
詳細な位置づけは `../concept_drift_existence/design.md` の §1.3 / §9 を参照。

## ②着手時に検討するメモ（暫定）
- **SHAP 値も反復の中央値で安定化する方向**。①で各セルを `N_REPEATS` 回繰り返し中央値を採る（①の §6.2）のと
  同じ理屈で、1 回のモデル学習・1 回の SHAP 計算は層2振り分け・モデル乱数でブレる。各反復でモデル学習→SHAP
  計算し、(Change × 特徴) ごとに反復方向の中央値を取ると①と一貫する。
- ただし集約は中央値で良いが、**赤/緑モデルはベース値が違うため SHAP の絶対値の大小は直接比較しない**。
  「符号反転・相対順位の変化」を中央値で見る、が整合的（①の §7.3 の方針に沿う）。
